THREAT INTELLIGENCE REPORT — Volt Typhoon
Source: Microsoft Threat Intelligence (MSTIC), May 2023
Classification: TLP:WHITE (public disclosure)

ACTOR OVERVIEW
==============
Volt Typhoon is a Chinese state-sponsored threat actor active since mid-2021,
targeting US critical infrastructure sectors including communications, manufacturing,
utilities, transportation, construction, maritime, government, IT, and education.
Primary assessed intent: pre-positioning for disruption of US-Asia communications
during a future geopolitical crisis, particularly around Taiwan.

INITIAL ACCESS
==============
Initial access achieved via exploitation of internet-facing Fortinet FortiGuard
appliances. The actor extracted credentials from Active Directory accounts used
by these devices, then used those credentials to authenticate to other network
systems. No novel zero-days confirmed; existing CVEs in Fortinet products exploited.

To avoid attribution, all command-and-control traffic is routed through compromised
small office / home office (SOHO) network equipment:
  - ASUS routers
  - Cisco routers and firewalls
  - D-Link routers
  - NETGEAR appliances
  - Zyxel firewalls and VPN concentrators

This proxy layer makes network-based attribution and blocking extremely difficult.

POST-COMPROMISE TTPs (LIVING OFF THE LAND)
==========================================
Volt Typhoon relies almost exclusively on built-in Windows tools ("living off the
land binaries" / LOLBins), making detection via traditional signature-based tools
very difficult.

Credential Access:
  - Dumped LSASS process memory to obtain OS credential hashes
  - Used ntdsutil.exe to create AD installation media containing username/password hashes:
    ntdsutil.exe "ac i ntds" "ifm" "create full C:\temp\pro" q q

Discovery:
  - Enumerated file systems, running processes, network config via:
    cmd.exe, powershell.exe, wmic.exe, ping, ipconfig, net, netstat
  - Detected virtualization environments before acting (sandbox evasion)

Collection:
  - Harvested credentials from web browsers
  - Staged collected data in password-protected archives before exfiltration

Lateral Movement / Persistence:
  - Established internal proxies using:
    netsh interface portproxy add v4tov4 ...
  - Used WMI for remote execution (no PsExec — evades common detection)
  - Authenticated via valid credentials harvested earlier — traffic mimics
    legitimate user activity and evades user/entity behavior analytics

Command and Control:
  - Custom variant of Impacket toolkit for network protocol operations
  - Fast Reverse Proxy (FRP) — open-source tool — used to tunnel C2 traffic
  - All C2 routed through compromised SOHO devices; no dedicated C2 infrastructure

  Behavior:Win32/SuspRemoteCmdCommandParent.A
  Behavior:Win32/VSSAmsiCaller.A
  Behavior:Win32/WmicRemote.A
  Behavior:Win32/WmiprvseRemoteProc.B

HUNTING QUERIES (KQL / Microsoft Sentinel)
===========================================
Domain Controller Installation Media Creation:
  DeviceProcessEvents
  | where ProcessCommandLine has_all ("ntdsutil", "create full", "pro")

Internal Proxy Establishment:
  DeviceProcessEvents
  | where ProcessCommandLine has_all ("portproxy", "netsh")
    or ProcessCommandLine has_all ("wmic", "process call create", "v4tov4")

MITIGATIONS
===========
  - Enforce MFA with FIDO2 hardware security keys
  - Disable unnecessary management interfaces on network edge devices
  - Enable ASR rules: credential stealing from LSASS, block PSExec/WMI process creation
  - Configure LSASS Protected Process Light (PPL) on Windows 11
  - Enable Windows Defender Credential Guard
  - Audit SOHO network devices for unexpected outbound proxy behavior
  - Hunt for ntdsutil.exe usage outside of known maintenance windows

UNCERTAINTIES
=============
  - Full scope of compromised SOHO devices used as proxies is unknown
  - Additional custom tooling beyond FRP variants may exist
  - Actor may maintain long-term dormant access on networks pending geopolitical trigger
